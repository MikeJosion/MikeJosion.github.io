LeetCode-快慢指针

Created2026-01-01|Updated2026-01-01|LeetCode模板数组
|Post Views:

🧠 要求原地修改,且要”删除/筛选/移动”元素

一句话结论：
👉 四个问题:结构变不变(判断是否适用),什么情况可以保留,slow,fast
快慢指针只关心:哪些元素应该被保留,并把其按顺序写在前面,本质不是“删除”，而是用合法元素覆盖前缀区
在这类题目中，返回值定义了“有效区间”，返回值之外的内容，算法语义上等同于“不存在” ,所以不需要关心后面多余的部分,只需要确保返回自己需要的结果

⭐ 模板

slow = 0 // slow：下一个“保留元素”该放的位置

for fast = 0 到 n-1: // fast：扫描整个数组
if 当前元素满足“保留条件”:
nums[slow] = nums[fast] // 覆盖到前面
slow = slow + 1

return slow // slow 即“新数组长度”

核心四个问题
① 结构变不变？

❌ 不新建数组

✅ 通常要求保持剩余元素的「相对顺序」

✅ 原地修改

👉 满足 ⇒ 快慢指针

② 什么情况下「可以保留」？

👉 哪些元素，应该被放到数组前面？

③ slow 是谁？

slow 永远表示：
下一个“合法元素”应该放的位置

它不是值

它不是下标技巧

它是 “结果区的尾巴”

fast 是谁？

fast 永远负责：
扫描整个数组，一个都不放过

所以：

for (fast = 0; fast < n; fast++)

是 永远不变的

题意： 给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素。元素的顺序可能发生改变。然后返回 nums 中与 val 不同的元素的数量。

2. LeetCode 26. 删除有序数组中的重复项

题意： 给你一个 非严格递增排列 的数组 nums ，请你 原地 删除重复出现的元素，使每个元素 只出现一次 ，返回删除后数组的新长度。元素的 相对顺序 应该保持 一致 。然后返回 nums 中唯一元素的个数。

3. LeetCode 80. 删除有序数组中的重复项 II
