LeetCode 2. 两数相加

Created2025-12-16|Updated2025-12-20|LeetCode
|Post Views:
给你两个 非空 的链表，表示两个非负的整数。它们每位数字都是按照 逆序 的方式存储的，并且每个节点只能存储 一位 数字。
请你将两个数相加，并以相同形式返回一个表示和的链表。
你可以假设除了数字 0 之外，这两个数都不会以 0 开头。

❄️ 难点解析

1️⃣ 为什么第一个节点要「特殊处理」？

if (head == NULL) { head = tail = node; }

空链表时，tail 是 NULL，不能直接用。

2️⃣ 最高位还有进位，如何正确补节点？

<div class="cold-alert error">
    <strong>❌ 错误写法（早期常见）：</strong>
    <br>
    <code>if (l1 != NULL || l2 != NULL || carry > 0)</code>
</div>

<div class="cold-alert success">
    <strong>✅ 正确理解：</strong>
    <p>进位只可能在 while 循环结束后 产生</p>
    <ul>
        <li>出了 while：</li>
        <li>l1 != NULL ❌（已经遍历完）</li>
        <li>l2 != NULL ❌（已经遍历完）</li>
    </ul>
    <hr>
    <p><strong>👉 这两个判断已经没有任何意义了</strong></p>
    <p>✅ 正确判断：</p>
    <pre>if (carry > 0) { /* 补一个新节点 */ }</pre>
</div>

3️⃣ 为什么每次新建节点都要 next = NULL？

新节点的 next 必须显式设为 NULL，否则它可能指向一块“垃圾内存”。
malloc 只分配内存，不初始化，node->next 里一开始是 随机值。

🧠 核心知识点总结

✅ 链表遍历与「对齐技巧」

不同长度链表的处理方式：

int a = l1 ? l1->val : 0; int b = l2 ? l2->val : 0;

含义：

  • 链表短了 → 当作补 0
  • 不需要提前算长度
  • 一行代码解决对齐问题
<hr style="border-color: rgba(255,255,255,0.1); margin: 15px 0;">

<h4>🔐 链表指针安全三原则</h4>
<ul>
    <li>用之前先判空：<code>if (l1) l1 = l1->next;</code></li>
    <li>新节点一定初始化：<code>node->next = NULL;</code></li>
    <li>尾节点必须指向 NULL：<code>tail->next = NULL;</code></li>
</ul>
