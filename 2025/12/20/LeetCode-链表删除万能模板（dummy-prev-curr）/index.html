🧠 链表删除万能模板（dummy + prev + curr）

一句话结论：

👉 LeetCode 中 所有可能删除头节点的链表题
👉 本质上 都可以反向映射为同一个删除模板

一、为什么一定要有「万能模板」？

链表删除常见翻车点：

  • ❓ 头节点被删怎么办？
  • ❓ 连续删一段节点，prev 该不该动？
  • ❓ free 之后还能不能访问？
  • ❓ 不同题目写法完全不统一，越写越乱？

👉 根本原因只有一个：

单链表删除 = 前驱节点的接线问题

所以我们人为制造一个“永远安全的前驱”。

二、三大角色（必须先记住）

dummy -> prev -> curr -> curr->next -> …

rust
复制代码

指针 角色 一句话职责
dummy 哨兵节点 保住链表入口，统一头删
prev 接线员 指向最后一个“已确认保留”的节点
curr 探险家 当前正在判断删不删的节点

👉 核心口诀：

  • ✅ curr 每轮都要前进
  • ❌ prev 只有在 curr 被保留时才前进

三、链表删除「万能模板」（母模板）

后面所有题目，都是在这个模板里换“删除条件”

四、反向映射 ① —— LeetCode 19（删除倒数第 N 个节点）

**
🎯 本题的“删除条件”是什么？
👉 删除的是：倒数第 N 个节点**
但在真正删除时，你已经准确拿到了它的前驱节点

🔁 映射回模板的核心动作

👉 本质仍然是：

前驱节点跳过一个节点

🧠 为什么 dummy 仍然必需？
n == 链表长度 → 删除头节点

dummy 保证「头删」逻辑和普通删除一模一样

五、反向映射 ② —— LeetCode 82（删除所有重复元素）

**题意：

👉 重复的值 一个都不留 🎯 删除条件（映射点）**

**🔁 套回万能模板（区间删除）

👉 关键点：

连续删除时 prev 一步不动

删除结束后 只接一次

六、反向映射 ③ —— LeetCode 83（保留一个重复值）

题意：

👉 重复值可以留 一个

🎯 为什么 83 不用 dummy / prev？
头节点永远保留

删除的是 curr->next

🔁 本质仍然是模板的“特例”

👉 理解要点：

83 =
「dummy + prev 模板中，prev 永远等于 curr」

七、三题对照速查表（考前专用）

题号 dummy prev 删除类型
19 ✅ 必须 ✅ 必须 单点删除
82 ✅ 必须 ✅ 必须 区间删除
83 ❌ 不需要 ❌ 不需要 单点（后继）

八、考试速记版（3 分钟背完）

✅ 万能结构
rust
复制代码
dummy -> prev -> curr
✅ prev 什么时候动？
curr 被确认保留时

❌ prev 什么时候不动？
curr 被删除时

✅ 删除三步走

🧠 终极一句话
链表删除不是“删节点”，
而是“前驱节点跳过一段内存”。

📌 写在最后
如果你以后看到任何链表删除题，只问自己一句话：

“我现在有没有一个安全的前驱？”

没有 → dummy

有 → 套模板
