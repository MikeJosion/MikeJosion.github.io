LeetCode-判断是否存在重复访问状态

以141. 环形链表为例讲解

给你一个链表的头节点  head ，判断链表中是否有环。如果链表中存在环 ，则返回 true 。 否则，返回 false 。

示例 1：

输入：head = [3,2,0,-4]
输出：true

所有“判断是否存在环 / 是否重复访问状态”的问题，核心思路只有两类：

① 利用不同速度制造必然相遇（快慢指针）

快慢指针（Floyd 判环）

核心思想 😶‍🌫️

使用两个指针，通过相对速度判断是否存在环

适用场景

  • 链表判环
  • 状态是否进入循环

判环模板（C）

示例代码 💕

② 记录来过的状态（哈希表）

一、哈希表一句话定义

哈希表 = 用 O(1) 的时间，快速判断“某个东西以前出现过没有”

关键词只有两个：
快速 + 判重 / 查找

二、刷题中哈希表解决什么问题？

本质只有 3 类 👇
① 判重（最常见）数字是否出现过 字符是否重复 节点是否访问过

不重要说明（可折叠）

👉 用 Set

② 记录映射关系（值 ↔ 值）

数 → 次数

键 → 位置

节点 → 其他信息

👉 用 Map

③ 缓存 / 状态记录

DFS / BFS 防止重复走

图是否访问过

链表是否成环

👉 Set / Map 都可以

三、Set vs Map 怎么选？（刷题金句）

只关心“来没来过” → Set
关心“来过 + 附加信息” → Map

四、哈希表的时间复杂度（必须会）

操作 平均时间复杂度 直观理解
插入 O(1) 常数时间
查找 O(1) 常数时间
删除 O(1) 常数时间

📌 正是因为这个，刷题才大量用哈希表

五、链表判环 —— 哈希表模板（你已经在用了）

为什么能判环？

Set 里存的是 节点引用（地址）

再次走到同一个节点 → contains == true

六、为什么不能只存 val？

❌ 常见错误说明

错误原因：

  • 不同节点，val 可能相同
  • 判环关注的是 节点是否被重复访问
  • 而不是 节点值是否重复

👉 必须将 ListNode 本身存入哈希表

七、哈希表 vs 快慢指针（重要对比）

方案 是否改结构 空间复杂度 思维难度
哈希表 不改 O(n) ⭐
快慢指针 不改 O(1) ⭐⭐

📌 面试更偏好快慢指针
📌 新手 / 判重类问题优先考虑哈希表

八、什么时候应该想到哈希表？

当你在题目中看到以下关键词时，条件反射想到哈希表：

  • 是否出现过？
  • 是否重复？
  • 是否访问过？
  • 是否存在环？
  • 去重
  • 第一次 / 再次

九、刷题万能判断公式（必记）

题目在问：
“之前有没有出现过某种状态？”
→ 直接考虑使用哈希表
